# 二、一般项目开发流程（公司之间是有差异）
- 确定项目需求-----拿下一个项目
- 编写《需求说明书》----不涉及技术，只涉及业务需求
- 编写《概要设计说明书》----- 涉及技术的的宏观的内容，数据库设计，页面原型
- 编写《详细设计说明书》----- 相当于伪代码
- 编码阶段coding----根据《详细设计说明书》--- 单元测试
- 联测-----项目组内部的行为
- 测试组进行全面的专业测试----《测试报告》
- 上线（测试阶段）
- 维护和二次开发
